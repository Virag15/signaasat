## ğŸ¯ Best Practices

### 1. **Before Committing**
```bash
# Check what you're committing
git status

# Review changes
git diff

# Only commit necessary files
git add <specific-files>
```

### 2. **Never Commit**
- Secrets (.env files)
- Dependencies (node_modules)
- Build artifacts (*.min.*)
- OS files (.DS_Store)
- Editor configs (.vscode/)
- Backup files (*.backup)

### 3. **Always Commit**
- Source code
- Configuration files
- Documentation
- Build scripts
- .gitignore

### 4. **Use Meaningful Commit Messages**
```bash
# Good âœ“
git commit -m "feat: Add mobile hamburger menu"
git commit -m "fix: Resolve scroll performance issue"
git commit -m "chore: Update .gitignore"

# Bad âœ—
git commit -m "update"
git commit -m "fix bug"
git commit -m "changes"
```

### 5. **Commit Message Format**
```
<type>: <description>

[optional body]
[optional footer]

Types:
- feat:     New feature
- fix:      Bug fix
- docs:     Documentation
- style:    Formatting
- refactor: Code restructuring
- perf:     Performance improvement
- test:     Adding tests
- chore:    Maintenance
```

```bash
# Ensure no secrets in git
```

## ğŸš¨ Security Checklist

Before pushing to GitHub:
- [ ] No API keys in code
- [ ] No passwords in code
- [ ] No `.env` files committed
- [ ] No database credentials
- [ ] No private keys
- [ ] No access tokens

If you accidentally commit secrets:
```bash
# Remove from history (DANGEROUS!)
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch path/to/secret' \
  --prune-empty --tag-name-filter cat -- --all

# Force push (after making sure!)
git push origin --force --all
```

**Better**: Rotate the exposed secrets immediately!
